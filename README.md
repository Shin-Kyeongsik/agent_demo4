# 법인카드 분석 Skill 사용 프롬프트

이 프로젝트는 `data/`에 업로드된 법인카드 사용내역 파일을 분석하는 `$analyze-corporate-cards` Skill을 포함합니다.

아래 프롬프트 예시는 사용자가 Codex에게 바로 요청할 수 있는 유형입니다.

## 1. 입력 파일 확인

업로드된 파일, 시트, 컬럼, 행 수를 먼저 확인하고 싶을 때 사용합니다.

```text
$analyze-corporate-cards를 사용해서 data/에 어떤 법인카드 파일이 있는지 확인해줘.
```

```text
data/의 엑셀 파일 구조를 확인하고, 시트명과 컬럼명을 요약해줘.
```

## 2. 부서별 사용 현황

부서별 총액, 건수, 평균 사용액을 알고 싶을 때 사용합니다.

```text
$analyze-corporate-cards를 사용해서 부서별 법인카드 사용 총액, 건수, 평균을 알려줘.
```

```text
data/의 법인카드 사용내역에서 어느 부서가 가장 많이 썼는지 부서별로 정리해줘.
```

```text
2026-05-01부터 2026-05-31까지 부서별 총액, 건수, 평균 사용액을 요약해줘.
```

## 3. 검토가 필요한 거래 탐지

중복 결제, 분할 결제, 고액 거래, 주말 사용, 메모 누락 등 감사 후보를 찾을 때 사용합니다.

```text
$analyze-corporate-cards를 사용해서 검토가 필요한 법인카드 거래를 찾아줘.
```

```text
중복 결제 의심 건과 분할 결제 의심 건을 우선순위 순으로 정리해줘.
```

```text
고액 거래, 주말 사용, 메모 누락 거래를 중심으로 감사 후보를 뽑아줘.
```

## 4. 종합 분석 리포트

지출 요약과 검토 후보를 함께 보고 싶을 때 사용합니다.

```text
$analyze-corporate-cards를 사용해서 data/의 법인카드 사용내역 전체를 분석하고 리포트를 작성해줘.
```

```text
월별 법인카드 파일 전체를 대상으로 지출 현황과 이상 거래 후보를 종합 분석해줘.
```

## 5. 특정 파일 또는 시트 분석

전체 `data/`가 아니라 특정 파일이나 시트만 분석하고 싶을 때 사용합니다.

```text
data/demo_card_transactions_2026_05.xlsx 파일만 대상으로 부서별 사용 현황을 알려줘.
```

```text
원본데이터 시트만 사용해서 검토가 필요한 거래를 찾아줘.
```

## 6. 기준값을 조정한 분석

고액 거래 기준이나 분할 결제 기준을 바꿔서 보고 싶을 때 사용합니다.

```text
고액 거래 기준을 2,000,000원으로 잡고 검토 대상 거래를 찾아줘.
```

```text
분할 결제 의심 기준을 합계 1,000,000원 이상으로 잡고 후보를 정리해줘.
```

## 7. 행 단위 근거 확인

요약 결과가 아니라 실제 거래 ID, 원본 파일, 원본 행 번호까지 확인하고 싶을 때 사용합니다.

```text
상위 검토 후보 10건에 대해 거래ID, 원본 파일, 원본 행 번호까지 보여줘.
```

```text
finding_id F0102의 근거 거래들을 자세히 확인해줘.
```

## 8. 거래처 표기 통합 확인

같은 거래처가 여러 이름으로 기록되었는지 확인하고 싶을 때 사용합니다.

```text
거래처명이 다르게 적힌 동일 거래처 후보를 찾아줘.
```

```text
CloudBox, CLOUD BOX, 클라우드박스처럼 같은 거래처로 보이는 항목을 통합해서 요약해줘.
```

## 권장 프롬프트 패턴

가장 안정적인 요청 형식은 다음과 같습니다.

```text
$analyze-corporate-cards를 사용해서 [분석 대상]에서 [보고 싶은 항목]을 [조건]에 맞춰 정리해줘.
```

예시:

```text
$analyze-corporate-cards를 사용해서 data/의 법인카드 사용내역에서 부서별 총액, 건수, 평균을 2026년 5월 기준으로 정리해줘.
```

```text
$analyze-corporate-cards를 사용해서 data/의 법인카드 사용내역에서 검토가 필요한 거래를 찾고, 중복 결제 의심 건을 우선해서 설명해줘.
```
