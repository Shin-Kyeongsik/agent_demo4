# 법인카드 검토 규칙

이 참조 문서는 기준값을 조정하거나, 탐지 로직을 설명하거나, 새로운 회사의 법인카드 정책에 맞게 스크립트를 수정해야 할 때만 읽는다.

## 필요 컬럼

스크립트가 최소한으로 요구하는 컬럼은 거래일자와 금액이다. 다음 컬럼이 함께 있으면 분석 품질이 높아진다.

- 거래 ID 또는 승인 번호
- 부서
- 사용자 또는 카드 소지자
- 직급 또는 직위
- 거래처 또는 가맹점
- 거래내용 또는 사용 목적
- 결제수단
- 메모, 비고, 적요

`거래ID`, `거래일자`, `부서`, `사용자`, `직급`, `거래처`, `거래내용`, `금액`, `결제수단`, `메모` 같은 한국어 원본 컬럼은 자동으로 표준 컬럼에 매핑한다.

## 기본 검토 규칙

- 정확 중복 후보: 거래일자, 부서, 사용자, 정규화된 거래처, 거래내용, 금액이 모두 같은 거래가 2건 이상 존재하면 검토 후보로 본다.
- 분할 결제 후보: 같은 거래일자, 부서, 사용자, 정규화된 거래처, 거래내용 조합이 2건 이상이고 금액이 서로 다르며 합계가 500,000원 이상이면 검토 후보로 본다.
- 고액 거래: 거래 금액이 1,500,000원과 해당 파일의 P95 금액 중 더 큰 값 이상이면 검토 후보로 본다.
- 재승인 또는 재처리 후보: 메모에 `재승인`, `재처리`, 승인 확인과 관련된 표현이 있으면 검토 후보로 본다.
- 메모 누락: 메모가 비어 있으면 사용 목적이나 증빙 상태가 확인되지 않는 거래로 본다.
- 주말 사용: 거래일자가 토요일 또는 일요일이면 검토 후보로 본다.
- 거래처 표기 통합 후보: 여러 원본 거래처명이 하나의 정규화 거래처명으로 매핑되면 확인 대상으로 본다.

모든 탐지 결과는 확정된 위반이 아니라 검토 후보로 표현한다.

## 카테고리 분류 기준

- `travel`: 호텔, 숙박, 리조트, 출장, 택시, 교통, 열차, 주차
- `meals`: 식대, 식사, 다과, 커피, 음료, 도시락, 분식, 한식
- `software`: SaaS, 클라우드, 협업툴, 보안 솔루션, 구독료, 라이선스, 데이터 분석 도구
- `education`: 교육, 세미나, 컨퍼런스, 연수, 리더십, 직무 교육
- `marketing_print`: 인쇄, 배너, 리플렛, 홍보물, 출력, 제작, 디자인
- `office_supplies`: 문구, 사무용품, 복사용지, 토너, 파일철

분류는 정책 위반 판단이 아니라 요약과 우선순위 산정을 돕기 위한 보조 신호다.

## 기준값 조정

분석 범위가 작거나 특정 부서만 볼 때는 기준값을 낮춘다. 임원 보고나 요약 리포트처럼 중요한 항목만 보고 싶을 때는 기준값을 높인다.

권장 시작값:

- 전사 월간 점검: 고액 거래 1,500,000원, 분할 결제 합계 500,000원
- 부서 단위 점검: 고액 거래 1,000,000원, 분할 결제 합계 300,000원
- 임원 요약: 고액 거래 2,000,000원, 분할 결제 합계 1,000,000원

## 해석 원칙

- 중복 후보는 취소, 재승인, 정정, 합법적인 반복 구매 가능성을 함께 확인한다.
- 분할 결제 후보는 승인 한도 회피 가능성과 실제 별도 구매 가능성을 함께 확인한다.
- 고액 거래는 승인권자, 예산 소유자, 업무 목적, 증빙 파일을 함께 확인한다.
- 주말 사용은 출장, 행사, 교육, 긴급 업무와 연결되는지 확인한다.
- 메모 누락은 영수증, 참석자 명단, 사전 승인, 업무 목적 설명을 요청한다.
- 거래처 표기 통합 후보는 월별 또는 부서별 비교 전에 같은 거래처로 묶어야 하는지 확인한다.
